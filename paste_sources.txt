# WHY THIS FILE EXISTS
# Forums and social media can't be reliably auto-scraped (login walls, blocked
# APIs, rate limits). So you collect these by hand from PUBLIC pages — just
# reading like any normal visitor — and paste them here. The engine reads this
# file and treats each line as a review, classified the same way as app-store
# reviews.
#
# FORMAT (one entry per line):
#       rating | source | the post text
#
#   - rating  = 1 to 5 if the platform shows one. Forums/social usually DON'T,
#               so leave it blank like this:   | forum | text here
#               (a blank rating is fine — the classifier handles it)
#   - source  = forum   OR   social   OR   reddit
#   - then | and the post text on ONE line
#
# Lines starting with # are ignored, so these notes are safe to leave in.
#
# HOW MANY: aim for ~15-25 per source. Quality over volume. Grab a RANGE —
# include a few neutral/positive/off-topic ones too, so your sample isn't
# biased toward your hypothesis. The classifier will sort relevance for you.
#
# WHERE TO COLLECT (all public, no login needed to read):
#   forum  -> community.spotify.com (Spotify's own forum), Quora "Spotify
#             discovery" answers, public Reddit threads read in your browser
#   social -> public X/Twitter posts searching "Spotify discover weekly",
#             "Spotify same songs", "Spotify recommendations boring"
